Panels are pre-built components that combine multiple elements into organized, reusable interfaces. They provide common patterns for displaying information, controls, and content in voice applications.

## Available Panels

## Common Features

All panels share common characteristics:

- **Panel Styling**: Built on the Panel component for consistent appearance
- **Collapsed Mode**: Many panels support a collapsed/compact display mode
- **Pipecat Integration**: Automatically integrate with Pipecat Client SDK
- **Customizable**: Support custom styling and configuration options
- **Responsive**: Adapt to container sizes and screen dimensions

## Usage Patterns

Panels are designed to be used in layouts like:

- **Sidebars**: Info panels, event logs
- **Main Content Areas**: Conversation panels, video displays
- **Control Areas**: Device controls, status displays
- **Dashboard Layouts**: Multiple panels arranged in grids

## Integration Requirements

Most panels require:
- `PipecatClientProvider` context for SDK integration
- `ConversationProvider` context (for conversation-related panels)
- Proper container sizing (panels typically need height constraints)

See individual panel documentation for specific requirements.
